Mark's Use Case and Interaction Flow

Use Case

Mark wants to find a board game to play this Christmas at his in-laws. He wants it to be fun, easy to explain, and fast. Mark is new to board games and wants help from more experienced gamers. So, he wants to create a forum where people can give him ideas.

precondition: Mark logs into his user account.

postcondition: Mark gets feedback and buys a game on Amazon.

Interaction Flow

  1. Mark clicks on the Geeklist menu item
  2. The site presents a drop down menu
  3. Mark clicks Create New
  4. The site presents a new page Create/Edit Geeklist
  5. Mark enters a Title and Description and clicks Save & Continue
  6. The site presents a new page with: This list has not been submitted
  7. Mark clicks Submit List
  8. Site presents the newly created Geeklist
